## Build Your Portfolio

### What a Technical Writing Role Entails

- [GitLab Technical Writing Handbook](https://handbook.gitlab.com/handbook/product/ux/technical-writing/) — How GitLab organizes technical writing work.
- [GitLab Documentation Style Guide](https://docs.gitlab.com/development/documentation/styleguide/) — Example of professional docs standards.

### Portfolio Project Ideas

- Create a small docs-as-code site using Docusaurus, MkDocs, or GitHub Pages.
- Write a REST API documentation sample with OpenAPI YAML, endpoint reference pages, error codes, and example requests/responses.
- Document an open-source project by improving a README, tutorial, troubleshooting page, or installation guide.
- Create a release notes sample and changelog.
- Create a knowledge base sample with task-based articles.
- Create a before-and-after rewrite showing how you improved clarity, structure, and readability.

## Removed or Replaced from the Original List

The following types of links were removed or replaced to keep the resource list reliable:

- Deprecated newsletter links from Revue.
- Podcast links that pointed to Stitcher, which is no longer an active podcast platform.
- Expired or unstable Slack/Discord invite links.
- Non-responsive job-board or newsletter links.
- Old course-year links replaced with current course homepages.
- Old documentation URLs replaced with current official documentation pages.
- Links without a protocol, such as `canva.com`, corrected to full `https://` links.
- Unstable or questionable book-download links replaced with official publisher, author, or free legal alternatives.

## Contribution Guidelines

When adding a new resource:

1. Prefer official documentation, active community sites, current course pages, or reputable blogs.
2. Avoid temporary invite links unless they are maintained by the official community.
3. Use the current canonical URL, not a redirect when possible.
4. Add a short description explaining why the link is useful.
5. Remove or replace links that return errors, redirect to unrelated pages, or point to deprecated services.

Happy learning and writing!
