# Style Guides

Use these to make your documentation consistent, plain, inclusive, and easy to maintain.

- [Microsoft Writing Style Guide](https://learn.microsoft.com/en-us/style-guide/welcome/) — Technology writing style and terminology.
- [Google Developer Documentation Style Guide](https://developers.google.com/style) — Style guide for developer documentation.
- [GitLab Documentation Style Guide](https://docs.gitlab.com/development/documentation/styleguide/) — Documentation standards for GitLab.
- [Red Hat Technical Writing Style Guide](https://stylepedia.net/) — Technical writing style reference.
- [Red Hat Supplementary Style Guide](https://redhat-documentation.github.io/supplementary-style-guide/) — Product documentation guidance.
- [A List Apart Style Guide](https://alistapart.com/about/style-guide/) — Web publication style guide.
- [Gatsby Style Guide](https://www.gatsbyjs.com/contributing/gatsby-style-guide/) — Style guide for open-source docs and content.
- [freeCodeCamp Developer News Style Guide](https://www.freecodecamp.org/news/developer-news-style-guide/) — Publication style guide.
- [How to Write a Style Guide as a Technical Writer](https://wonexo.hashnode.dev/how-to-write-a-style-guide-as-a-technical-writer) — Practical article by Segun Olumide.
